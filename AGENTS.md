## Project Overview
This project implements and analyzes **Distinction Centrality** ($\beta_i$), a social network metric theoretically inspired by Pierre Bourdieu's theory of agonistic competition in social fields. It models the trade-off between status (influence/ eigenvector centrality) and constraint (average status of neighbors in the node-deleted subgraph).

The mathematical formulation is:
$$\beta_i = \alpha s_i - \kappa_i$$
where $\alpha$ is a scalar adjustment factor:
$$\alpha = \frac{\bar{\mathbf{\kappa}}}{\bar{\mathbf{s}}}$$

## Core Functions Directory

All core custom metrics and modeling scripts are located in the `Functions/` folder:

- **`Functions/distinction.R`**: Calculates distinction centrality ($\beta_i$), scaled distinction, status ($s_i$), constraint ($\kappa_i$), and the scalar factor $\alpha$ for a given `igraph` object.
- **`Functions/tab.cent.R`**: Formats the centrality scores using `kableExtra` and saves them as LaTeX tables inside the `Tabs/` folder.
- **`Functions/plot.graph.norm.R`**: Generates a standard `ggraph` plot of the network colored by the distinction level of each node (Red = highest distinction, Blue = positive, Purple = neutral, Orange/Tan = negative).
- **`Functions/connect.caves.R`**: Caveman graph generation and random edge additions.
- **`Functions/make.broker.cliques.R`**: Generates brokerage structures over clique subgraphs.
- **`Functions/make.broker.trees.R`**: Generates brokerage structures over tree subgraphs.
- **`Functions/make-bonacich.R`**: Generates networks for Bonacich (2007) comparisons.
- **`Functions/plot.graph.cat.R`**: Categorical visualization of distinction scores.

## Helper Scripts (`Code/`)

- **`Code/diss.status.R`**: Computes status using similarity-discounted adjacency matrix.
- **`Code/make.estrada.R`**: Generates Estrada (2005) graph examples.

## Quarto Website & Rendering Notes

The project includes a Quarto website (configured in `_quarto.yml`). 
**Important Note on Rendering:** `analysis.qmd` is computationally heavy as it generates dozens of network plots and tables simultaneously. 
- Attempting to add complex HTML/flexbox layouts or using raw `cat()` to format these outputs for the web makes the Quarto render times prohibitively slow. 
- When assisting with the website, keep Quarto outputs standard and minimal. Prefer relying on Quarto's built-in caching (`cache: true` in execute options) if heavy structural changes are needed.
