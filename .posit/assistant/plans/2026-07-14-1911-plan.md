# Plan: Separate Website Rendering from Data Analysis

## The Problem
Currently, the root folder is configured as a Quarto `website` project. Because of this, running `quarto render` automatically discovers and tries to render every `.qmd` file in the folder, including heavy data analysis files like `analysis.qmd` and `jilbert.qmd`. 

## The Solution
Instead of moving files into different folders or manually maintaining a `.quartoignore` list of things to exclude, we can use an explicit **include list** in your Quarto configuration. 

By adding a `render` key to `_quarto.yml`, we can tell Quarto exactly which files constitute the website. Any file not on that list will simply be ignored by the site builder, allowing them to coexist peacefully in the root folder.

### Steps to Implement:

1. **Update `_quarto.yml`:**
   Add a `render` array to the `project` section specifying only the website files:
   ```yaml
   project:
     type: website
     output-dir: _site
     render:
       - index.qmd
       - functions.qmd
       - examples.qmd
   ```

2. **Clean up `.quartoignore`:**
   Remove the explicit ignores for `analysis.qmd` and `jilbert.qmd` that we added previously, as they will no longer be necessary. The explicit `render` list handles it much more cleanly.

3. **Resulting Workflow:**
   * **To build the website:** Run `quarto render` (or hit the render button on `index.qmd`). It will instantly build *only* those three website pages.
   * **To run an analysis:** Open `analysis.qmd` or `jilbert.qmd` and run them interactively, or run `quarto render analysis.qmd` directly. It will compile your LaTeX tables and plots without trying to shove them into the website.
